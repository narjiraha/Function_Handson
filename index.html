Q.4/ "Print output: var x = 21; girl (); console.log(x) function girl() { console.log(x); var x = 20; };" 

        ANS
        Undefined

        Explanation:The code defines a global variable x with the value 21. Then, it defines a function girl that logs a variable x to the
        console. Inside the function, a new variable x is declared with the value 20, but it only has function scope and is not
        the same as the global variable x. When the function is called and logs the variable x, the local variable x has not yet
        been assigned a value, so it logs undefined to the console.

Q.9/ "Guess the output Abc(); const Abc = function(){ let value = 20; console.log(value); }" 

        ANS:
        Error

        Abc();
        const Abc = function () {
        let value = 20;
        console.log(value);
        };

        Explanation:
        This code will result in an error because the function "Abc" is declared using a function expression which is assigned
        to a variable using the "const" keyword.
        
        When the program starts running, the "Abc()" function is called before it is declared. However, function expressions are
        not hoisted like function declarations, meaning that they cannot be called before they are defined. So it will through an error.
